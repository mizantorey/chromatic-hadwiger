Phase 1
Initialization — Optimal k-Coloring
Start with a graph G that has chromatic number χ(G) = k. Compute an optimal proper k-coloring — assign each vertex a color 1 through k such that no two adjacent vertices share a color.

The color classes are:
A₁, A₂, …, Aₖ ⊆ V(G)
where Aᵢ = all vertices with color i. These are independent sets — no edges inside any Aᵢ.

★ Key: By Definition 3.5 (Color-Class Immutability), vertices in Aⱼ will always remain in branch set Bⱼ throughout the algorithm. The color classes are permanent anchors.
Phase 1
Initialize Branch Sets: Bᵢ ← Aᵢ
Set each branch set equal to its color class:
Bᵢ(0) = Aᵢ   for all i = 1, …, k
The free vertex set is initially empty:
F(0) = V(G) \ S(0) = ∅
since every vertex starts in some branch set.

At this point: Corollary 3.6 already guarantees that every edge between Aⱼ and Aₗ in G is an edge between Bⱼ and Bₗ in the contracted graph H(G, B) — so many inter-edges already exist.

⚡ Problem: Color class Aᵢ might be disconnected in G! Branch set Bᵢ must be connected (Definition 3.4). Phase 2 will fix this.
Phase 2
BFS Repair — Connecting Disconnected Branch Sets
Problem: Branch set Bᵢ may be disconnected (its color class Aᵢ is an independent set in G, so internal connectivity relies on paths through other vertices).

Solution — BFS Expansion:
Pick the expansion center cᵢ ∈ Aᵢ. Run BFS from cᵢ, absorbing free vertices (vertices in F(t) = V \ S(t)) to create a connected path to every other vertex in Aᵢ.
Bᵢ(t+1) = Bᵢ(t) ∪ {free path vertices}
Free vertex set shrinks:
F(t) = V(G) \ S(t)   (S = ∪ Bⱼ)

🔍 Corollary 3.6 is preserved: since free vertices have no color-class assignment, absorbing them never moves an Aⱼ-vertex into a foreign branch set.
Phase 2
After BFS Repair — All Branch Sets Connected
After iterating BFS repair for each branch set, every Bᵢ is now connected in G.

The contracted graph H(G, B) has k nodes. Many inter-edges between branch sets already exist (guaranteed by Corollary 3.6 — color-class immutability).

Remaining challenge: some Bᵢ might still be isolated in H(G, B) — no edges to any other branch set. This happens when BFS absorbed free vertices in a way that blocked inter-set connections.

Phase 3 handles exactly this case.
All Bᵢ: connected ✓
Corollary 3.6 preserved ✓
F(t) ⊆ V(G)

⚠ Some contracted nodes in H may have degree 0 (isolated). Phase 3 will add edges until H = K_k.
Phase 3
Alternating Connector Repair — Case 3a
Bci is isolated in H(G,B): no edges to any other branch set. We must add at least one edge.

Case 3a — Simple Move:
Find a neighbor vertex nb ∈ Bⱼ such that nb has a neighbor v ∈ Bci. If nb is NOT an articulation point of Bⱼ — i.e., removing nb from Bⱼ leaves Bⱼ still connected — then:
Move nb from Bⱼ → Bci
Result: Bci now has an edge to (the remaining) Bⱼ. |E(H)| increases by 1.

↗ Each iteration of Phase 3 increases |E(H)| by ≥ 1. Max iterations ≤ k(k−1)/2 (total edges in K_k). Algorithm always terminates.
Phase 3
Alternating Connector — Case 3b (The Transfer)
Case 3b — Articulation Point Transfer:
If nb IS an articulation point of Bⱼ — removing it would disconnect Bⱼ — we cannot move it directly. Instead, we transfer the edge:

nb is the sole connection Bⱼ → Bₗ for some Bₗ. Transfer: the edge (Bⱼ, Bₗ) becomes edge (Bci, Bₗ).
Edge transferred: Bⱼ–Bₗ → Bci–Bₗ
Net effect: |E(H)| stays same or increases. Bci gains an edge to Bₗ (no longer isolated). No edge is destroyed — only re-routed through Bci.

Remark 8.10: Under Definition 3.5 (strict color-class immutability), Phase 3 is theoretically redundant — but ensures computational completeness and handles all edge cases explicitly.
